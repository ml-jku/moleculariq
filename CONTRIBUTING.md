# Contributing to MolecularIQ

Thank you for your interest in contributing to MolecularIQ! This document provides guidelines for contributing to the MolecularIQ project.

## Where to Contribute

The MolecularIQ project is organized into multiple repositories. Please direct your contributions to the appropriate repository:

### [moleculariq-benchmark](https://github.com/ml-jku/moleculariq-benchmark)
- New task definitions
- Dataset improvements
- Verifier implementations
- Documentation for task creation

### [moleculariq-lm-eval](https://github.com/ml-jku/moleculariq-lm-eval)
- Evaluation pipeline improvements
- Model adapter implementations
- Performance optimizations
- Integration with new models

### [moleculariq-leaderboard](https://github.com/ml-jku/moleculariq-leaderboard)
- UI/UX improvements
- Visualization features
- Submission flow enhancements
- Deployment optimizations

### This Repository (moleculariq)
- General documentation improvements
- Cross-repository coordination
- Overall project organization
- Community discussions

## How to Contribute

1. **Open an Issue**: Discuss your proposed changes before implementing
2. **Fork the Repository**: Create your own fork of the relevant repository
3. **Create a Branch**: Use descriptive branch names (e.g., `feature/new-task`, `fix/evaluation-bug`)
4. **Make Changes**: Follow the coding standards and guidelines
5. **Test**: Ensure your changes don't break existing functionality
6. **Submit a Pull Request**: Provide a clear description of your changes

## Coding Standards

- Follow PEP 8 for Python code
- Include docstrings for all public functions and classes
- Add type hints where applicable
- Write clear commit messages
- Add tests for new features

## Reporting Issues

When reporting issues, please include:
- A clear description of the problem
- Steps to reproduce
- Expected vs. actual behavior
- Environment details (OS, Python version, etc.)
- Relevant error messages or logs

## Code of Conduct

We are committed to providing a welcoming and inclusive environment. Please:
- Be respectful and considerate
- Accept constructive criticism gracefully
- Focus on what's best for the community
- Show empathy towards others

## Questions?

- Open a [Discussion](https://github.com/ml-jku/moleculariq/discussions) for general questions
- Check existing issues before creating new ones
- Reach out to maintainers for complex contributions
